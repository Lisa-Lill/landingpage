Release Notes 08/07/2016
  • Replaced the old Map View component (using the Flash-based GViewer) with the new D3.js-based interactive Map View, compatible with touch gestures
  • Added new QTLNetMiner instance for chickpea
  • Updated the force-directed layouts for KNETviewer
  • KNETviewer now allows users to "Export JSON" to export their network for usage in Cytoscape desktop
  • Query Suggestor now dynamically provides suggestions as users type in queries

Release Notes 10/09/2015
  • Replaced the Ondex Web Java-applet with the new KNETviewer, a cytoscapeJS-based lightweight Network View component
  • Added new QTLNetMiner instances for Brassica Oleracea and >Maize
  • Gene View now allows users to choose between Known and Novel Targets
  • Query Suggestor enhanced: For every search term (keyword), the Query Suggestor now provides the top 25 suggestions per concept type
  • Added a template to dynamically update all release notes with detailed statistics whenever the knowledge networks are updated
  • KNETviewer: Added multiple dynamically-rendered layout options, implemented via force-directed layout algorithms
  • KNETviewer: Added sliding panels to KNETviewer window to display addition information
  • KNETviewer: Added 'Item Info' pane to KNETviewer window to display concept/ relation information
  • KNETviewer: Added highlighting of term(s) in concept/ relation name and Item Info pane, if they match user's query term(s)
  • KNETviewer: Added feature to show/ hide concepts or relations individually or multiple (by type) at a time
  • KNETviewer: Added 'Show concept neighborhood' feature
  • KNETviewer: Added 'search by name' feature
  • KNETviewer: Network graphs can now be exported as images as JSON datasets

Release Notes 05/09/2014
  • Changed URL of all QTLNetMiner instances to https://
  • Updated the way the applet is embedded (.jnlp file) to meet new standards
  • Improved client performance (especially under IE) by definiing a javascript event handler
  • Added new buttons to enlarge, download and share the network in the Network View
  • Added a button to select all user target genes in the Gene View
  • Fixed 5sec search delay that occurred when performing first search or after session had expired
  • New feature to display top N genes in the Gene View (not fixed to top 100 any longer)
  • Sort Evidence View table by tf-idf score of a document
  • Added example queries (under ? of search) which automatically fill entire search form
  • Added a message to notify when the backend service is down

Release Notes 12/11/2013
  • Support for species without a sequenced genome; based on unigenes (e.g. wheat)
  • Support for multiple organisms in one QTLNetMiner instance (e.g. potato, tomato etc.)
  • Full support of Lucene query syntax
  • New query suggestion wizard for building advanced queries and including synonymous terms into the search. It gets triggered iwhen user expands the query suggestion box.
  • New real-time messaging system to feedback the number of genes and documents that will be found while user is typing the query
  • New tab Evidence View to display an evidence centric view of the search results including information about the evidence types, #genes linked to the evidence, lucene score, etc
  • Linked Evidence View to Network View by showing a network of all genes related to the selected evidence
  • Linked PMIDs in the Evidence View and evidence boxes to PubMed
  • Extend the evidence column of the Gene View with the list of evidence names when clicking on the evidence types
  • New buttons to exclude specific evidence concepts shown in the Gene and Evidence views from the search
  • Support for NOT statements in the query to exclude any text or particular Ondex concepts from the search
  • Show in brackets the number of documents containing the query terms (regardless whether linked to genes or not)
  • Use Lucene for highlighting concepts in Ondex network view that contain the query terms
  • Notify when the server is down
  • Get the number of genes within user specified QTL from the server instead of parsing the gene2postion file. This file is not required anymore.
